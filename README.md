# UMATSoilPlasticityAgent 🤖
We developed **UMATSoilPlasticityAgent**, an LLM agent developed to automate the generation of stress integration algorithms for small-strain, rate-independent elastoplastic soil models, with direct applicability to **ABAQUS UMAT** subroutine development.

By integrating a retrieval-augmented generation (RAG) module, a generation-review loop (GRL) module and a formatted prompt paradigm, this tool transforms structured constitutive model descriptions into high-fidelity numerical implementations.

## 🌟 Key Features

- **Automated UMAT Generation**: Bridges the gap between complex constitutive equations and numerical implementation.
- **RAG**: Leverages a specialized database of geotechnical UMAT exemplars to ensure mechanical consistency and high-fidelity code synthesis.
- **GRL**: Employs an iterative "Generator-Reviewer" workflow to refine code based on syntax and computational logic.


## 🚀 Getting Started

### 1. Prerequisites
- An active **OpenAI API Key** (configured in `config.py`).
- Required libraries: `openai`, `langchain`, `ipywidgets`.

### 2. Initialization (First-Time Use)
To build the local vector database from the provided UMAT library:
1. Open the Jupyter Notebook `test_display.ipynb`.
2. **Uncomment** the following line to initialize the FAISS index:
   ```python
   # build_db_from_code_files("./data")
   ```
3. Run the cell. A folder named `faiss_db` will be generated in your current directory.

### 3. Running the Agent
1. Once the database is initialized, re-comment the initialization line.
2. Execute the notebook to launch the interactive UI.
3. Input your constitutive model parameters (Yield surface, Plastic potential surface, etc.) following the provided prompt template.


## 🛠 Configuration
- **LLM Backend**: The default model is `GPT-4.1`. You can switch model versions or adjust hyperparameters (e.g., temperature) in `llm.py`.
- **API Credentials**: Ensure your OPENAI_API_KEY is correctly set in `config.py`.
- **Knowledge Expansion**: You can easily expand the agent's expertise by adding your own UMAT files to the `UMAT database` folder and rebuilding the index.
